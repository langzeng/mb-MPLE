This repository provides the code necessary to reproduce selected results from the manuscript "Mini-batch Estimation for Deep Cox Models: Statistical Foundations and Practical Guidance." Figures 1–3 are fully reproducible using the provided code. Figures 4 and 5 require access to the AREDS dataset, which is available upon request from the dbGaP online repository (accession: phs000001.v3.p1).

"simulation_cox_reg_result.ipynb" reproduces Figure 2 using simulation results stored in the "Results" folder. These results can be regenerated using "simulation_cox_reg.py".

"simulation_population_gradient.R" generates Figure 1b through simulation.

"LinearScalingRule.ipynb" generates Figure 3 through simulation.

"func_acc.R" and "func_acc.py" define custom functions required by the scripts.

"ACC_form_Libraries_and_dependencies.txt" documents the libraries and dependencies of the scripts.

"AREDS.ipynb" contains the code used to generate Figures 4 and 5. However, it is not executable without access to the AREDS dataset due to privacy restrictions. The dataset can be requested from dbGaP (accession: phs000001.v3.p1). It relies on six processed AREDS datasets (intermediate files), including
start_stop_working.Rdata
start_stop_test.Rdata
x_train_demo_scaled_7087.npy
x_test_demo_scaled_778.npy
img_train_base_7087.npy
img_test_base_778.npy.
Here are the descriptions of those files:
start_stop_working.Rdata and start_stop_test.Rdata contain time-to-event data for 7,087 and 778 patients, respectively. These datasets were generated using areds1_data_processing_acc.R, which merges two preprocessed AREDS data sources: (1) longitudinal clinical visit data (longitudinal_master_pheno.txt), produced by areds_longitudinal.py from the raw AREDS data, and (2) image-level data (master_pheno_jpg.txt), generated by master_pheno_jpg.py using the raw AREDS data and available image filenames. After merging these two datasets, areds1_data_processing_acc.R constructs the data in a time-to-event (start–stop) format. All three data preprocessing code files are located in the "Code_ACC/AREDS_Preprocessing".
x_train_demo_scaled_7087.npy, x_test_demo_scaled_778.npy, img_train_base_7087.npy and mg_test_base_778.npy contain the baseline covariates for 7087 and 778 patients, respectively. The first two are the low-dimensional features scaled within [0,1]. These covariates can be directly extracted from start_stop_working.Rdata and start_stop_test.Rdata and are stored in a scaled tensor format for simplicity. The last two files are the baseline fundus images of patients in tensor format.
